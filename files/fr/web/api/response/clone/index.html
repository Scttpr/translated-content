La méthode clone() de l'API {{domxref("Response")}} attribue à une variable un clone identique en tout point d'un objet réponse.

clone() renvoie une exception {{jsxref("TypeError")}} si le {{domxref("Body")}} de la réponse a déjà été utilisé. La raison principale de l'existance de la méthode clone() est qu'elle permet une utilisation multiple d'objets {{domxref("Body")}} (seulement quand ils sont à usage unique).

Synthaxe

var response2 = response1.clone();

Paramètres

Aucun.

Valeur

Un objet {{domxref("Response")}}.

Exemple

Dans notre example d'utilisation de la méthode clone de l'API Reponse (voir la version en ligne) nous créons un nouvel objet {{domxref("Request")}} en utilisant {{domxref("GlobalFetch.fetch","fetch()")}}. Quand la requête est résolue avec succès; nous la clonons et extrayons un blob de chacune des deux réponses grâce à deux appels {{domxref("Body.blob")}} créant ensuite un objet URL à partir des blobs via la méthode {{domxref("URL.createObjectURL")}} pour finalement les afficher dans deux éléments {{htmlelement("img")}} distincs pour finalement les afficher dans deux éléments {{htmlelement("img")}} distincs.. 

var image1 = document.querySelector('.img1');
var image2 = document.querySelector('.img2');

var myRequest = new Request('flowers.jpg');

fetch(myRequest).then(function(response) {
  var response2 = response.clone();

  response.blob().then(function(myBlob) {
    var objectURL = URL.createObjectURL(myBlob);
    image1.src = objectURL;
  });

  response2.blob().then(function(myBlob) {
    var objectURL = URL.createObjectURL(myBlob);
    image2.src = objectURL;
  });
});
